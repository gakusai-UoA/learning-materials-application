# 学習アプリ 要件定義兼仕様書

## 1. アプリケーション概要

勉強会用の各種資料（JavaScript/TypeScript/Hono/React等）を、実際に手を動かしながら体感・学習できる専用のデスクトップアプリケーション。勉強会開始前に配布し、参加者が各自のPC（Windows/Mac）で操作することを想定する。

- **利用形式:** 事前配布型ハンズオンアプリ
- **動作環境:** Windows / macOS
- **ベース技術:** Electron

### 1.1 スタート画面（メニュー画面）

- アプリ起動直後に表示される画面。
- 学習したいパート（Part-0 〜 Part-9）を選択できる。
- 各パートのボタンやカードには、「どの技術スタックを使っているか（解説箇所）」を小さめの文字などでわかりやすく表示する。
- 順番通りの学習だけでなく、任意のパートへのジャンプも用意にする。

## 2. 画面レイアウト・インタフェース

画面は左右分割等のペイン構造とし、一画面で「学ぶ」と「試す」を完結させる。

- **左ペイン: スライド表示領域**
    - アプリ内にプリインストールされたPDFスライドを1ページ1スライドとして表示。
    - 下部に「←」「→」ボタン（またはキーボードの矢印キー）を配置し、ページネーションを行う。
    - アニメーション効果は不要。
- **右ペイン: エディタ ＆ プレビュー/実行領域**
    - **エディタ:** Monaco Editor を組み込み、コーディングを行う。
    - **トグル機能:** 「エディタビュー」と「ブラウザ（プレビュー）ビュー」を切り替え可能とする。
    - **ブラウザビュー:** `localhost:{PORT}` （例: `localhost:8787`）などを表示し、内部のリンク操作やクエリパラメータの変更（例: `/a?p=1`）などを可能にする。

## 3. コア機能と動作定義

### 3.1. バックグラウンドプロセスの実行・セットアップ

- **環境自動構築:** アプリ起動時に、ユーザーのシステムに `pnpm` がインストールされているかを確認する。未インストールの場合はバックグラウンドで自動導入（例: `npm install -g pnpm` や PowerShell/sh 経由での導入スクリプト）を行い、環境構築のハードルを下げる。
- ユーザーの記述内容や学習進度にあわせ、裏側で `pnpm dlx vitest`、`pnpm exec wrangler`、Node.js のローカルサーバー等を起動・制御する。
- 実行状態やログ（標準出力など）を適切にキャッチし、必要に応じて右ペイン（または専用のコンソール領域）に表示する。

### 3.2. 学習コンテンツの構成

本アプリは以下の勉強会用資料（Markdown/PDF）に特化して構成する。各パートはスタート画面から選択でき、学習画面内でも前のパート・次のパートへ簡単に移動できるナビゲーションを備える。

1. **Git:** `Part-0`
2. **JavaScript/TypeScript:** `Part-1`, `Part-2`, `Part-3`
3. **Hono:** `Part-4`, `Part-5`, `Part-8`
4. **React:** `Part-6`, `Part-7`, `Part-8`
5. **デプロイ技術 / 発展:** `Part-9` （Firebase Hosting や Cloudflare Workers(wrangler) 等の実際のデプロイ機能やビルド確認をエミュレート、またはローカルから実行連携できるようにする）

### 3.3. ファイルの保存と永続化

- **形式:** 内部保存ではなく、ユーザPC上の特定のローカルディレクトリ（テンプレートプロジェクト）と直接同期・上書き保存する。
- **目的:** 勉強会終了後、ユーザがそのままVSCode等で対象フォルダを開き、テンプレートとしてシームレスに開発を継続できるようにするため。

### 3.4. コーディングサポートの制限

- 初学者向けであるため、**Biome.js等によるインラインのエラー表示（赤い波線など）は行わない。** （Monaco EditorとBiomeの共存・連携はさせない）。
- 一方でプロジェクト自体のソースコードフォーマット（Electronアプリ自体のソースコード）としては設定通りBiomeを使用し、品質を維持する。

## 4. プロジェクト構成・技術スタック

本アプリ専用に特化した設計とし、ディレクトリやファイルは役割ごとに細かく分割して管理する。

### 4.1. 使用技術

- **デスクトップアプリ:** Electron
- **フロントエンド:** React
- **スタイリング:** Tailwind CSS v4, shadcn/ui
- **エディタ:** Monaco Editor
- **フォーマッタ/リンタ（開発者用）:** Biome.js
- **パッケージ管理:** pnpm (Electron内でのコマンド実行もpnpmベース)
- **ビルドツール:** Vite (electron-vite), electron-builder (AppImage等の出力用)

### 4.2. アセット管理

- スライド用のPDFは `assets/` などのディレクトリ内に事前配置し、バンドルして配布する。

## 5. 制限事項・特記事項

- **専用アプリ:** 本アプリケーションは今回の勉強会専用の手順とコンテンツに特化して構築し、過度な汎用化は行わない。
- **クイズについて:** クイズ機能は本アプリ内には実装せず、別途Slido等を利用する想定。
